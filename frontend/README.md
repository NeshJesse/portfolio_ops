# Portfolio Frontend

A modern, responsive portfolio website built with Next.js 16, TypeScript, and Tailwind CSS. This frontend is designed to work seamlessly with the Portfolio-OPs CLI tool for automated project discovery and data generation.

## Features

- **Data-Driven**: All content is loaded from JSON and Markdown files in `/public/portfolio-data/`
- **Responsive Design**: Mobile-first approach with Tailwind CSS
- **Modern Stack**: Next.js 16 with App Router, TypeScript, and React 19
- **Admin Dashboard**: Built-in content management system for editing portfolio content
- **Project Curation**: Toggle featured projects directly from the web interface
- **Markdown Support**: Full markdown rendering with syntax highlighting for README files
- **Project Showcase**: Dynamic project grid with filtering, search, and categorization
- **Server Actions**: Next.js server actions for seamless data updates
- **SEO Optimized**: Static generation for all pages
- **Accessibility**: WCAG compliant components

## Data Sources

The portfolio reads from the following files in `/public/portfolio-data/`:

- `projects.json` - Project data (generated by Portfolio-OPs CLI)
- `about.md` - About page content
- `skills.json` - Skills and technologies
- `experience.json` - Work experience
- `contact.json` - Contact information
- `assets/` - Project screenshots and logos

## Getting Started

### Prerequisites

- Node.js 18+ 
- npm or yarn

### Installation

```bash
# Install dependencies
npm install

# Start development server
npm run dev
```

Open [http://localhost:3000](http://localhost:3000) to view the portfolio.

### Building for Production

```bash
# Build the application
npm run build

# Start production server
npm start
```

## Development

### Adding New Components

Components are located in `/components/` and follow these patterns:

- Use TypeScript for type safety
- Implement responsive design with Tailwind CSS
- Follow accessibility best practices
- Use semantic HTML elements
- Server components by default, use `'use client'` only when needed

### Data Loading

Data loading utilities are in `/lib/`:

- `projects.ts` - Project data loading, filtering, and categorization
- `content.ts` - Markdown and JSON content loading (server/client compatible)
- `utils.ts` - Helper functions (date formatting, color schemes, search, sorting, stats)

### Server Actions

Server actions are defined in `/app/actions.ts`:

- `getProjects()` - Read all projects
- `toggleProjectFeatured()` - Toggle featured status
- `getAboutData()`, `updateAboutData()` - About content management
- `getSkillsData()`, `updateSkillsData()` - Skills management
- `getExperienceData()`, `updateExperienceData()` - Experience management

### Admin Dashboard

The admin dashboard (`/home/*`) provides:

- **Project Curation** (`/home/curate`) - Select featured projects
- **Content Editing** - Forms for About, Skills, and Experience
- **Server-side Updates** - Changes persist to JSON files
- **Path Revalidation** - Automatic cache invalidation on updates

### Styling

The project uses Tailwind CSS with custom configurations:

- Custom color schemes for different technologies
- Responsive design utilities
- Custom prose styles for markdown content
- Consistent spacing and typography

## Integration with Portfolio-OPs CLI

This frontend is designed to work with the Portfolio-OPs CLI tool:

1. **Data Generation**: CLI scans your projects and generates `projects.json`
2. **Asset Management**: CLI copies screenshots and logos to `assets/`
3. **Incremental Updates**: CLI only updates changed projects for speed
4. **Automated Workflow**: CLI can watch for changes and auto-update

### Workflow

```bash
# 1. Generate portfolio data
python3 portfolio.py generate

# 2. Start development server
npm run dev

# 3. Watch for changes (optional)
python3 portfolio.py watch
```

## Key Features

### Public Pages

- **Homepage**: Hero section, about preview, skills showcase, experience timeline, featured projects
- **Projects Page**: Filterable grid with search, category/language/tag filters, sorting options
- **Project Details**: Full README rendering, screenshot galleries, git stats, tech badges
- **About/Skills/Experience/Contact**: Dedicated pages for portfolio sections

### Admin Dashboard

- **Project Curation**: Visual interface to mark projects as featured
- **Content Management**: Edit About, Skills, and Experience content through forms
- **Real-time Updates**: Changes reflect immediately with Next.js revalidation

### Technical Features

- **Type Safety**: Full TypeScript coverage with comprehensive type definitions
- **Server Actions**: Secure server-side data mutations

- **Responsive Design**: Mobile-first with Tailwind CSS
